## Recommended Next Small Win

Best next item for `v2.2`:

- add a small evaluation CLI that loads checkpoints and prints comparable metrics without editing scripts

Why this is the best small/high-return task:

- it is much smaller than algorithmic work
- it directly helps with the repo's current pain point: people can run training, but they still struggle to judge whether a checkpoint is actually better
- it improves reproducibility immediately by standardizing how checkpoints are compared
- it creates a clean place to report:
  - profit vs random
  - profit vs checkpoint pools
  - zero-sum / invalid-state counts
  - number of completed hands

Good scope for this branch:

- new script: `scripts/evaluate_models.py`
- accepts one or more checkpoints
- runs a fixed number of games with fixed seeds
- prints a compact table of comparable results
- optionally writes JSON or CSV
